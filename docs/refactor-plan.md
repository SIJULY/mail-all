# 重构计划

## 总目标
在不改变功能、不改变 UI、不改变路由和 SMTP 行为的前提下，将大型单文件 `app.py` 分阶段拆分为多文件项目。

## 硬性约束
1. 功能不变
2. 不丢功能
3. 不新增功能
4. UI 不变
5. 路由、参数名、表单字段、返回行为尽量不变
6. SMTP 行为不变
7. 所有代码必须从源码迁移
8. 未读取源码不迁移、不改动
9. 分阶段执行
10. 每一步可验证、可回滚

## 当前阶段划分

### 阶段 0：重构前盘点
- 目标：获得结构信息
- 当前状态：已完成
- 产物：`app_scan.json`

### 阶段 1：建立项目框架
- 目标：只创建目录、占位文件、记录文件
- 不做：不迁移业务代码，不修改 `app.py`
- 验证：结构存在，`app.py` 未改动

### 阶段 2：建立函数/路由/模块映射表
- 目标：根据源码逐步确认哪些代码迁往哪里
- 前提：必须先读取相关源码
- 不做：不猜测实现

### 阶段 3：先迁移低风险基础模块
- 目标：优先迁移源码中已确认的基础逻辑
- 例：配置、常量、数据库连接、设置读写
- 前提：必须逐段读取源码后再迁移

### 阶段 4：迁移 SMTP / 消息处理
- 目标：迁移 SMTP、MIME、消息处理等
- 约束：SMTP 行为保持不变

### 阶段 5：迁移路由与 UI 相关代码
- 目标：将路由和页面逻辑拆分归类
- 约束：UI 不变、路由不变、字段不变

### 阶段 6：收口入口结构
- 目标：必要时整理 `app/__init__.py`
- 前提：只有在已确认安全时才进行

## 当前明确不做的事情
- 不重写整个项目
- 不做业务优化
- 不擅自模板化
- 不改交互
- 不改接口契约
- 不改 SMTP 协议行为
